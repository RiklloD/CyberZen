# Environment Variables Reference

Full reference for every environment variable CyberZen reads. Variables split into two groups by where they're configured:

- **Frontend** (`VITE_*`) — set in `apps/web/.env.local`, bundled by Vite, visible in the browser
- **Backend** (everything else) — set in Convex via `bunx convex env set VAR_NAME value`, never exposed to the browser

Variables marked **Required** will cause a hard failure if unset. Everything else fails gracefully and disables the related feature.

---

## How to Set Variables

### Frontend (Vite)

```bash
# apps/web/.env.local
VITE_CONVEX_URL=https://<slug>.convex.cloud
VITE_APP_TITLE=CyberZen
```

### Backend (Convex)

```bash
bunx convex env set GITHUB_TOKEN ghp_...
bunx convex env set SLACK_WEBHOOK_URL https://hooks.slack.com/...
# etc.
```

### List currently configured backend vars

```bash
bunx convex env list
```

## Core

| Variable | Where | Required | Default | Description |
|---|---|---|---|---|
| `CONVEX_DEPLOYMENT` | `.env.local` | **Yes** | — | Convex deployment identifier. Format: `dev:<slug>` or `prod:<slug>`. Generated by `bunx convex dev`. |
| `VITE_CONVEX_URL` | `.env.local` | **Yes** | — | Convex backend URL. Format: `https://<slug>.convex.cloud` |
| `CONVEX_SITE_URL` | `.env.local` | No | — | Convex HTTP Actions base URL. Format: `https://<slug>.convex.site` |

## GitHub

| Variable | Where | Required | Default | Description |
|---|---|---|---|---|
| `GITHUB_TOKEN` | Convex | No | — | Personal access token (PAT) for GitHub API calls — repo list, SBOM fetch, advisory queries. Scopes needed: `repo`, `read:packages`. |
| `GITHUB_PR_TOKEN` | Convex | No | `GITHUB_TOKEN` | Separate token for PR creation if you want to restrict the primary token's write access. Falls back to `GITHUB_TOKEN`. |
| `GITHUB_SECURITY_ADVISORY_TOKEN` | Convex | No | `GITHUB_TOKEN` | Token for querying the GitHub Security Advisory Database. Falls back to `GITHUB_TOKEN`. |
| `GITHUB_WEBHOOK_SECRET` | Convex | No | — | HMAC-SHA256 secret for validating incoming GitHub webhooks. Generate with `openssl rand -hex 32`. |
| `GITHUB_ISSUES_REPO` | Convex | No | Finding's own repo | Override where GitHub Issues are created. Format: `owner/repo`. |
| `GH_TOKEN` | Convex | No | — | Fallback token, read by the GitHub CLI conventions. Used if `GITHUB_TOKEN` is unset. |
| `GHES_BASE_URL` | Convex | No | — | GitHub Enterprise Server base URL. Example: `https://github.acme.com`. Leave unset for github.com. |
| `GHES_API_URL` | Convex | No | `{GHES_BASE_URL}/api/v3` | GHES REST API URL override. Defaults to the standard GHES path if `GHES_BASE_URL` is set. |

## Summary

| Category | Count | Notes |
|---|---|---|
| Core / Convex | 3 | Required — app won't start without these |
| Frontend (Vite) | 4 | Optional except `VITE_CONVEX_URL` |
| API Security | 5 | Set `SENTINEL_API_KEY` before production |
| Source Control | 8 | GitHub only needed for repo integration |
| CI/CD Webhooks | 5 | One per CI system you connect |
| Notifications | 8 | Slack + Teams + PagerDuty + OpsGenie |
| Issue Trackers | 14 | Jira + Linear + Shortcut |
| AI / Semantic | 3 | Set `OPENAI_API_KEY` for full breach correlation |
| Vuln Databases | 4 | `NVD_API_KEY` improves rate limits |
| Observability | 9 | Datadog + Prometheus + SIEM |
| Internal Services | 2 | Python services only |
| **Total** | **73** | |

**Minimum to run locally:** `CONVEX_DEPLOYMENT` + `VITE_CONVEX_URL`

**Minimum for a useful production deployment:**
- `SENTINEL_API_KEY` — secure your operator endpoints
- `GITHUB_TOKEN` — repo access and SBOM ingestion
- `GITHUB_WEBHOOK_SECRET` — validate webhook payloads
- One notification channel (`SLACK_WEBHOOK_URL` or `TEAMS_WEBHOOK_URL`)
- One issue tracker (`JIRA_*` or `LINEAR_*`)
- `NVD_API_KEY` — avoid NVD rate limiting
- `OPENAI_API_KEY` — semantic breach correlation
